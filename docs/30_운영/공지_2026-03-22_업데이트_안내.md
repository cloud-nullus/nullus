## 2. OIDC Provider 추상화 (PR #4)

Keycloak 외에 **Authentik**도 OIDC provider로 사용할 수 있도록 추상화했습니다.

**환경변수 하나로 전환 가능:**

```bash
VITE_OIDC_PROVIDER=keycloak    # 또는 authentik
```

- **프론트엔드**: `oidc-providers.ts`에서 provider별 role 추출 자동 분기
- **백엔드**: `OIDCProvider` 인터페이스 + Keycloak/Authentik 구현체, JWT 미들웨어 주입
- **Mock 모드**: `VITE_AUTH_MODE=mock`으로 OIDC 서버 없이 개발 가능 (기존과 동일)

새 provider를 추가하려면 `docs/20_개발가이드/Nullus_OIDC_Provider_가이드.md`를 참고하세요.

- 백엔드: `PATCH /api/v1/stacks/:id/tools` (Clean Architecture)

## 참고: 환경변수 변경

`web/.env.example`에 아래 항목이 추가되었습니다.

```bash
VITE_AUTH_MODE=mock              # mock | oidc
VITE_OIDC_PROVIDER=keycloak     # keycloak | authentik
VITE_OIDC_AUTHORITY=http://localhost:8180/realms/nullus
VITE_OIDC_CLIENT_ID=nullus-web
```

기존 개발 환경에서는 변경 없이 동작합니다 (`VITE_AUTH_MODE=mock` 기본).
